1. pipe - The pipe operator is not an operator itself, but it's the mechanisim that allows you to chain operators together. Think of it as a pipeline where your data flows through a series of pure functions. You'll see it in every example from now on.

2. map - It works just like the array map method. It applies a function to each value emitted by the source Observable and emits the result.

3. filter - It emits a value form the source only if it passes a certain condition you provide. It's essential for selecting the data you care about.

4. first - The first operator emits only the first value from a stream. You can also provide a predicate function to find the first value that matches a certain condition.
